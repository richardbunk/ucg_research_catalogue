# RCG-004 — UCG Research Catalogue v1.0 Sponsor Directive Set

**Status:** Adopted  
**Applies To:** UCG Research Catalogue v1.0  
**Role Context:** Executive Sponsor / Repository Architect working method

---

## Purpose

These directives define the minimal command vocabulary used by the Executive Sponsor to manage repository construction while delegating editorial authority to the Repository Architect.

The Repository Architect is expected to continue work independently within the approved scope until a defined stopping condition is encountered.

---

## Project-Level Directives

### Initialize Project

> Begin Version 1.0 publication.

Starts the approved publication roadmap.

### Resume Project

> Resume Version 1.0.

Continue from the current project state. No additional explanation is required.

### Continue

> Continue.

Proceed from the current task. Do not restate the plan unless necessary.

### Proceed to Next Phase

> Proceed to Phase X.

Advance to the next approved publication phase.

### Complete Current Phase

> Complete the current phase.

Continue working until the phase is complete or a stopping condition is encountered.

---

## Publication Directives

### Publish

> Publish.

Draft, review, normalize, publish, update indexes, and continue within the approved scope.

### Publish Batch

> Publish the current batch.

Complete the entire approved publication batch.

### Finish Governance

> Complete Governance.

Publish all remaining governance documents.

### Finish Infrastructure

> Complete Infrastructure.

Publish manifests, registers, templates, and repository infrastructure.

---

## Research Directives

### Process Intake

> Process this intake.

Determine source classification and execute the appropriate workflow.

### Continue Processing

> Continue processing.

Resume from the current research pipeline.

### Populate Catalogue

> Continue catalogue population.

Continue processing approved source material.

---

## Review Directives

### Status

> Status.

Summarize current phase, completed work, remaining work, blockers, and next action.

### Review

> Review current work.

Perform a consistency review without introducing architectural changes.

### Freeze

> Freeze.

Lock the current publication state and record future observations for Version 1.1.

---

## Approval Directives

### Approved

> Approved.

Proceed without further review.

### Sponsor Decision

Used only when the Repository Architect requests guidance.

---

## Stopping Conditions

The Repository Architect should interrupt delegated progress only when:

- architectural ambiguity exists,
- repository philosophy must change,
- project scope changes,
- connector failure prevents publication,
- or the approved phase is complete.

All other editorial decisions remain delegated.

---

## Standing Delegation

Unless otherwise instructed:

- architecture remains frozen,
- Version 1.0 conventions remain fixed,
- editorial authority is delegated,
- improvements are recorded for Version 1.1,
- repository construction remains subordinate to reconstruction of UCG vNext.

The Executive Sponsor's responsibility is to maintain direction.

The Repository Architect's responsibility is to execute the work.

---

## Primary Operating Directive

The preferred short-form command for delegated work is:

> Continue until completion or blocker.

When this directive is issued, the Repository Architect shall continue through the approved scope without pausing for routine confirmations and shall return only when a stopping condition is encountered or the approved scope is complete.
